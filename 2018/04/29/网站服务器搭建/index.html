网站服务器搭建：遇到的问题及解决方法

发表于 2018-04-29 | 更新于 2019-05-14 | 分类于 实用技术 | 评论数： | 阅读次数：
本文字数： 968 | 阅读时长 ≈ 1 分钟

总结和回顾前段时间搭建网络服务器时遇到的问题和解决方法。

服务器环境：Windows Server 2008

  目标服务器是Windows Server 2008 系统，且服务器上已有apache和MYSQL服务，但是因为我需要PHP服务，又不想单独安装PHP（太麻烦），就直接用WAMP搭建了。

  这中间遇到一些问题，记录下来以备查阅。也希望能帮助到和我一样的懒人。

Wampserver 版本问题

  安装最新版 Wampserver 时一直会报错，提示缺少依赖包。于是我把Visual C++ Redistributable for Visual Studio 2008、2010、2015系列都装了一遍，也没解决问题，并且安装2015时又有其他报错，烦不胜烦。

解决方案： 我怀疑 Windows Server 2008 对最新的WAMP的某些组件不支持，所以最终选择了Wampserver 2.5 版本安装，成功运行。

有时候版本不一定越新越好，新版本遇到问题时不妨尝试一下原先的版本。

Wampserver 端口切换

  Wampserver 自带端口切换功能，可以把MYSQL、Apache等端口直接切换。但在切换MYSQL端口后会发现PHP涉及到数据库的功能不能正常用了。

解决方案：更改 php.ini 中 mysql.default_port 的值为新的MYSQL端口值。

有时候还是不能过分依赖这些便捷软件，自己一定要知道配置文件在哪，都有什么东西，该怎么配。

设置为允许外网访问

  Wampserver 自带切换离在线状态，但切换完之后发现还是不能在其他电脑上访问该服务器。

解决方案： 这不是 Wampserver 的问题，而是Windows防火墙的问题。在防护墙中设置入站规则，增加相应的端口。

总结：

  1. 有些问题看上去很容易，真正实施起来却会遇到各种各样的问题。
  2. 问题解决后回头看解决方案会发现非常简单，但真正难的是如何定位问题，尤其是搭建一个系统的时候。这需要对整个系统非常了解，对事件流程非常了解，才能逐步排查直到找到错误。
  3. 信息检索能力是一种非常宝贵的能力。（有个好的搜索引擎也很重要）
